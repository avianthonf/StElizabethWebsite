<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Desktop menu DOM → Browser hover engine | Hover state drives mega-menu visibility; rapid mouse movements can queue animation frames |
| Mobile overlay DOM → Body scroll | RemoveScroll must completely lock body scroll while overlay is open; failure leaks scroll through menu |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Spoofing (rapid hover spam) | Mega-menu open/close cycle | mitigate | 100ms hoverIntent delay prevents dropdown flicker; AnimatePresence presence-aware exit prevents double-animate |
| T-01-04 | Tampering (z-index overlap) | Mobile menu z-index 10000 vs subsequent modals | accept | Current z-index scale is Walker-parity (header 9999, mobile 10000). Future modals must declare z-index > 10000 or use portal. Documented in CONCERNS.md as known limitation |
| T-01-05 | Elevation of privilege (Escape key closes wrong overlay) | Global keydown listener | mitigate | Escape handler checks mobileOpen state before closing; uses window-level listener with cleanup on unmount; no other overlay registers Escape on window |
</threat_model>
